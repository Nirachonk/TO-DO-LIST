# TO DO LIST

เว็บแอปจัดการงาน (Work Order Tracker) — บันทึกงาน แนบรูป กำหนดวันรับงาน/วันครบกำหนด เลือกประเภทงาน (มิเตอร์ / บริการ / อื่นๆ) และระดับความสำคัญ (สำคัญ / ปกติ) พร้อม Dashboard สรุปภาพรวม

- **หน้าบ้าน (Frontend):** HTML/CSS/JS ล้วน ๆ ไม่ต้อง build รันบน GitHub Pages ได้ทันที
- **หลังบ้าน (Backend):** Google Apps Script — เก็บข้อมูลงานใน Google Sheet และเก็บรูปภาพใน Google Drive

ไม่มีค่าใช้จ่ายใด ๆ ทั้งสิ้น ใช้บัญชี Google ปกติของคุณ

---

## โครงสร้างไฟล์

```
todo-list-app/

├── apps-script/
│   └── Code.gs              <- โค้ด backend (วางใน Google Apps Script)
└── README.md
```

## ขั้นตอนที่ 1: ตั้งค่า Google Sheet + Apps Script (Backend)

1. ไปที่ [sheets.google.com](https://sheets.google.com) แล้วสร้างสเปรดชีตใหม่ 1 ไฟล์ ตั้งชื่อว่า `TO DO LIST Database`
2. ในสเปรดชีต ไปที่เมนู **ส่วนขยาย (Extensions) > Apps Script**
3. จะเปิดหน้าต่างแก้ไขโค้ดขึ้นมา ให้ลบโค้ดตัวอย่าง (`function myFunction() {}`) ออกให้หมด
4. เปิดไฟล์ `apps-script/Code.gs` จากโปรเจกต์นี้ คัดลอกโค้ดทั้งหมด แล้ววางแทนที่
5. กด **บันทึก** (ไอคอนรูปแผ่นดิสก์ หรือ Ctrl+S)
6. กดปุ่ม **Deploy > New deployment**
7. กดไอคอนรูปเฟือง ⚙️ เลือกประเภทเป็น **Web app**
8. ตั้งค่าดังนี้:
   - **Execute as:** Me (บัญชีของคุณ)
   - **Who has access:** Anyone
9. กด **Deploy** ระบบจะขอให้อนุญาตสิทธิ์ (Authorize) — กดอนุญาตให้ครบทุกขั้นตอน (จะมีหน้าเตือน "Google hasn't verified this app" ให้กด Advanced > ไปที่โปรเจกต์ (ไม่ปลอดภัย) เพราะเป็นสคริปต์ของคุณเอง)
10. หลัง Deploy สำเร็จ จะได้ **Web app URL** หน้าตาประมาณ:
    ```
    https://script.google.com/macros/s/XXXXXXXXXXXXXXXXX/exec
    ```
    **คัดลอก URL นี้เก็บไว้** จะใช้ในขั้นตอนถัดไป

> ระบบจะสร้างชีตชื่อ `Tasks` และโฟลเดอร์ Google Drive ชื่อ `TODO_LIST_Uploads` ให้อัตโนมัติในการเรียกใช้งานครั้งแรก ไม่ต้องสร้างเอง

### หากแก้โค้ดภายหลัง
ทุกครั้งที่แก้ไข `Code.gs` ต้องกด **Deploy > Manage deployments > แก้ไข (ไอคอนดินสอ) > Version: New version > Deploy** ใหม่ ไม่เช่นนั้นการเปลี่ยนแปลงจะไม่มีผล

## ขั้นตอนที่ 2: เชื่อม Frontend เข้ากับ Backend

1. เปิดไฟล์ `docs/index.html`
2. หาบรรทัดนี้ (อยู่ท้ายไฟล์ ในส่วน `<script>`):
   ```js
   const GAS_URL = "PASTE_YOUR_GOOGLE_APPS_SCRIPT_WEB_APP_URL_HERE";
   ```
3. แทนที่ด้วย Web app URL ที่ได้จากขั้นตอนที่ 1 เช่น:
   ```js
   const GAS_URL = "https://script.google.com/macros/s/XXXXXXXXXXXXXXXXX/exec";
   ```
4. บันทึกไฟล์

---

## ขั้นตอนที่ 3: อัปโหลดขึ้น GitHub และเปิดใช้งาน GitHub Pages

1. สร้าง repository ใหม่บน GitHub (public หรือ private ก็ได้)
2. อัปโหลดไฟล์ทั้งหมดในโฟลเดอร์นี้ขึ้น repository (หรือใช้คำสั่ง git):
   ```bash
   git init
   git add .
   git commit -m "TO DO LIST website"
   git branch -M main
   git remote add origin https://github.com/<username>/<repo-name>.git
   git push -u origin main
   ```
3. ไปที่ repository บน GitHub > **Settings > Pages**
4. ในหัวข้อ **Build and deployment**:
   - Source: **Deploy from a branch**
   - Branch: **main** และเลือกโฟลเดอร์เป็น **/docs**
5. กด **Save** รอสักครู่ (1-2 นาที) GitHub จะให้ลิงก์เว็บไซต์ เช่น:
   ```
   https://<username>.github.io/<repo-name>/
   ```
6. เปิดลิงก์นี้ — เว็บไซต์ TO DO LIST พร้อมใช้งาน 🎉

## ฟีเจอร์ปฏิทิน & แจ้งเตือน (ใหม่)

เมนู **🗓️ ปฏิทิน** ในแถบด้านซ้าย เปิดหน้าปฏิทินแยกต่างหากจากรายการงาน สำหรับบันทึกนัดหมาย เช่น การประชุม หรือเรื่องสำคัญที่ต้องจำ:

- **+ เพิ่มนัดหมาย**: กรอกหัวข้อ เลือกประเภท (ประชุม / เรื่องสำคัญ / อื่นๆ) วันที่ เวลา รายละเอียด และเลือกติ๊ก "แจ้งเตือนเมื่อถึงเวลา" ได้
- **ปฏิทินรายเดือน**: วันที่มีนัดหมายจะมีจุดสีใต้ตัวเลข คลิกวันที่เพื่อดูนัดหมายเฉพาะวันนั้นในช่อง "นัดหมายที่จะถึง"
- **แถบแจ้งเตือนวันนี้**: ถ้าวันนี้มีนัดหมาย จะมีแถบสีชมพูขึ้นเตือนทั้งในหน้า Dashboard และหน้าปฏิทิน
- **การแจ้งเตือนแบบเรียลไทม์**: ถ้าติ๊ก "แจ้งเตือนเมื่อถึงเวลา" ระบบจะเช็คทุก 30 วินาที และแจ้งเตือนแบบ Popup (Notification) เมื่อถึงเวลานัดหมาย **เงื่อนไข: ต้องเปิดหน้าเว็บไซต์นี้ค้างไว้ในเบราว์เซอร์** (เป็นข้อจำกัดของเว็บไซต์แบบ static ที่ไม่มีเซิร์ฟเวอร์ส่งแจ้งเตือนเบื้องหลัง จึงไม่สามารถแจ้งเตือนได้ขณะปิดหน้าเว็บ/ปิดเครื่อง)
- ข้อมูลนัดหมายจะถูกเก็บในชีตใหม่ชื่อ `Events` ในสเปรดชีตเดียวกัน (สร้างอัตโนมัติ)

> หากต้องการแจ้งเตือนแม้ปิดหน้าเว็บไซต์ (เช่น ส่ง LINE/อีเมลล่วงหน้า) จะต้องเพิ่ม Google Apps Script Trigger แบบ time-based ฝั่ง backend เพิ่มเติม ซึ่งนอกเหนือขอบเขตของเวอร์ชันนี้ — แจ้งได้หากต้องการให้พัฒนาเพิ่ม

## การใช้งานเว็บไซต์

- **+ เพิ่มงานใหม่**: กรอกชื่องาน รายละเอียด เลือกประเภทงาน (มิเตอร์ / บริการ / อื่นๆ) ระดับความสำคัญ (สำคัญ / ปกติ) วันที่รับงาน วันครบกำหนด แนบรูปภาพได้
- **คลิกที่การ์ดงาน (ใบสั่งงาน)**: เปิดดูรายละเอียดงานแบบเต็ม แก้ไข ทำเครื่องหมายว่าเสร็จแล้ว หรือลบงานได้
- **แท็บด้านบน**: กรองงานตามประเภท (ทั้งหมด / มิเตอร์ / บริการ / อื่นๆ)
- **ตัวกรองด้านขวา**: กรองตามความสำคัญ สถานะ และค้นหาชื่องาน
- **Dashboard ด้านบน**: สรุปจำนวนงานทั้งหมด งานที่ยังไม่เสร็จ งานที่เสร็จแล้ว งานเกินกำหนด และงานสำคัญ

ข้อมูลทั้งหมดจะถูกบันทึกลง Google Sheet ของคุณโดยตรง (ชีตชื่อ `Tasks`) และรูปภาพจะถูกเก็บในโฟลเดอร์ Google Drive ชื่อ `TODO_LIST_Uploads` — เปิดดู/สำรองข้อมูลได้ตลอดเวลาจาก Google Sheets และ Google Drive ของคุณเอง

## หมายเหตุด้านความปลอดภัย

- Web app URL ของ Apps Script ทำหน้าที่เหมือน API สาธารณะ (Who has access: Anyone) เพื่อให้เว็บไซต์แบบ static เรียกใช้งานได้โดยไม่ต้อง login — ใครก็ตามที่มี URL นี้จะสามารถอ่าน/เขียนข้อมูลในชีตได้ เหมาะสำหรับใช้งานส่วนตัว/ทีมเล็ก ไม่ควรเผยแพร่ URL นี้ในที่สาธารณะ
- หากต้องการความปลอดภัยเพิ่มเติม (ระบบล็อกอิน แบ่งสิทธิ์ผู้ใช้) จะต้องพัฒนาเพิ่มเติมจากโครงสร้างนี้

## แก้ปัญหาเบื้องต้น

| อาการ | สาเหตุที่เป็นไปได้ |
|---|---|
| ขึ้นข้อความ "ยังไม่ได้เชื่อมต่อ Google Apps Script" | ยังไม่ได้แก้ค่า `GAS_URL` ในไฟล์ `index.html` |
| กดบันทึกงานแล้วไม่มีอะไรเกิดขึ้น / ค้าง | ตรวจสอบว่า Deploy เป็น **Anyone** สามารถเข้าถึงได้ ไม่ใช่ "Only myself" |
| รูปภาพไม่ขึ้น | ตรวจสอบสิทธิ์แชร์ไฟล์ใน Google Drive โฟลเดอร์ `TODO_LIST_Uploads` ว่าเป็น "Anyone with the link" |
| แก้โค้ด Code.gs แล้วไม่มีผล | ต้องกด Deploy ใหม่ทุกครั้ง (New version) ตามขั้นตอนด้านบน |
